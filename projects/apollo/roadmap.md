# Apollo Roadmap

## Phase I, Foundation
**Goal:** Apollo can reliably ingest or use curated source material, preserve project structure, retrieve relevant evidence, and support grounded querying through one governing interface.

### Core concerns
- workspace and repo clarity
- clean project corpus
- retrieval viability
- project control continuity

### Phase I focus in practice
For the current cycle, Phase I is narrower than the total vision. It is focused on:
- using the existing running stack
- reusing the existing `Vault/Clean/` work
- proving the first real Apollo query loop

## Phase II, Analytical Core
**Goal:** Apollo can reason across sources rather than merely store them.

### Intended additions
- entity and claim extraction
- relationship mapping
- contradiction detection
- timeline workflows
- uncertainty-aware synthesis

## Phase III, Operational Core
**Goal:** Apollo becomes a real project/life assistant, starting with Aurelius.

### Intended additions
- regimen models
- schedule generation
- reminders
- adherence tracking
- outcome logging
- protocol revision support

## Phase IV, Program Modules
**Goal:** Apollo becomes a host platform for specialized workflows.

### Intended additions
- Aurelius as a mature operational module
- Remote Viewing tasking and feedback workflows
- subliminal/pathworking generation workflows
- module framework for future additions

## Phase V, Mature Orchestration
**Goal:** Apollo becomes a governable system of delegated intelligence.

### Intended additions
- specialist subagents where useful
- richer task delegation
- better observability and auditability
- stronger local/cloud routing policy

## Build order principle
The dependency chain should remain:

**Ingestion / clean corpus -> retrieval -> grounded query loop -> analysis -> operations -> modules -> orchestration**

## Anti-sprawl rule
Do not let later-phase ideas displace the current phase proof.
Apollo advances correctly when each phase leaves behind a simpler, more reliable foundation for the next one.
